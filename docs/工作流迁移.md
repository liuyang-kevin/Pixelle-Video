  af_scail 是什么？

  SCAIL (Spatially-Constrained Adversarial Learning)
  是一个动作迁移模型——输入一张人物图片 + 一段动作视频 +
  提示词，输出让图片中的人物"跳"视频里的动作。标题 API-SCAIL动作迁移，作者
  user_hzelb9wt，更新于 2026-03-23。

  移植到本地 selfhost 需要什么？

  移植后文件应放在 workflows/selfhost/af_scail.json。

  1. ComfyUI 自定义节点：

  ┌───────────────────────────────┬────────────────────────────────┐
  │             仓库              │              用途              │
  ├───────────────────────────────┼────────────────────────────────┤
  │ kijai/ComfyUI-WanVideoWrapper │ WanVideo 扩散模型 + SCAIL 节点 │
  ├───────────────────────────────┼────────────────────────────────┤
  │ kijai/ComfyUI-SCAIL-Pose      │ NLF 姿态估计 + RenderNLFPoses  │
  └───────────────────────────────┴────────────────────────────────┘

  2. 模型文件（需提前下载）：

  ┌─────────────────────────────────────┬──────────────────┬────────┐
  │                模型                 │       用途       │  大小  │
  ├─────────────────────────────────────┼──────────────────┼────────┤
  │ Wan2.1-I2V-14B (FP8)                │ 视频生成基础模型 │ ~15GB  │
  ├─────────────────────────────────────┼──────────────────┼────────┤
  │ SCAIL 权重                          │ 动作迁移         │ ~5GB   │
  ├─────────────────────────────────────┼──────────────────┼────────┤
  │ NLF (nlf_l_multi_0.3.2.torchscript) │ 3D 姿态估计      │ ~700MB │
  ├─────────────────────────────────────┼──────────────────┼────────┤
  │ ViTPose + YOLOv10m                  │ 人物检测+2D姿态  │ ~1GB   │
  ├─────────────────────────────────────┼──────────────────┼────────┤
  │ CLIP ViT-H/14                       │ 参考图像编码     │ ~2.5GB │
  ├─────────────────────────────────────┼──────────────────┼────────┤
  │ Wan VAE                             │ 编解码           │ ~500MB │
  └─────────────────────────────────────┴──────────────────┴────────┘

  3. 硬件要求： 24GB+ VRAM（推荐 FP8 量化版降低需求）

  4. 参数映射（Pixelle-Video 传过来的参数）：

  video  → VHS_LoadVideo 节点（参考动作视频）
  image  → LoadImage 节点（被迁移的人物图片）
  prompt → CLIPTextEncode 节点
  second → frame_load_cap（视频时长控制）
